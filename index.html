为什么是 DigitalOcean，不是 AWS

两年前我从 Heroku 搬出来，原因很简单——Heroku 免费层取消了，最便宜的计划跑一个简单 API 都要 $7/月，而且冷启动 10 秒。我需要一个便宜的、能同时跑多个项目的地方。

看了 AWS Lightsail、Vultr、Linode、DigitalOcean。最后选了 DO。不是因为性能最强——是因为文档最好、定价不会突然暴涨、不用半夜收到天价账单。

AWS 免费层超了之后，一个不注意就 $300。DO 的定价是固定的——$4 就是 $4，$6 就是 $6。对我这种不想当 DevOps 工程师的人来说，这是核心价值。

"DigitalOcean 不是性能最强的云——是最不可能半夜让你收到惊吓账单的云。"

一台 $6/月 Droplet（1GB RAM, 1vCPU, 25GB SSD），跑了：

两年内没有一次意外停机。有一次自己把 Nginx 配炸了，那不能怪 DO。

$6 + $15 = $21/月跑一个全栈 SaaS。对比之前 Heroku $25/月只跑一个后端，便宜且控制权大多了。
